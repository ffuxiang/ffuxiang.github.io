    参数编码与解码——源码笔记¶

    介绍¶

      在算法Faster R-CNN、RetinaNet中，实现边界框编码和回归参数解码的过程，假设x_a,y_a,w_a,h_a为锚点绝对坐标，x,y,w,h为物体边界框坐标，t_x,t_y,t_w,t_h为各个坐标数据的回归参数：

    • 边界框编码：边界框坐标->回归参数
    t_x=\frac{x-x_a}{w_a},t_y=\frac{y-y_a}{h_a},t_w=\log{\frac{w}{w_a}},t_h=\log{\frac{h}{h_a}}
    • 回归参数解码：回归参数->边界框坐标
    x=w_a*t_x+x_a,y=t_y*h_a+y_a,w=w_a*e^{t_w},h=h_a*e^{t_h}

    代码实现¶

    参考链接：

    • https://github.com/WZMIAOMIAO/deep-learning-for-image-processing
    • https://github.com/pytorch/vision/tree/master/torchvision/models/detection

    类初始化

    需要指明两个变量：

    • weights：回归参数的权重分布
    • bbox_xform_clip：回归参数的最大值

    注：

    • weights参数其实是用来对边框回归得到的偏移值做标准化的，以提高模型精度（默认偏移量均值为0，方差需要自己设置）。如果是一阶段目标检测，则都设置为1即可，如果是二阶段目标检测，则两个阶段的weights设置通常不同，因为第二阶段是基于第一阶段做预测，因此第二阶段方差更小，Faster R-CNN中第一阶段均为1，第二阶段设置为[10,10,5,5]。参考链接：https://blog.csdn.net/u014311125/article/details/117750396

    编码过程——encode

    传入：锚点坐标、边界框坐标

    返回：回归参数

    解码过程——decode

    传入：锚点坐标、回归参数

    返回：边界框坐标

    class BoxCoder(object):
    """
    This class encodes and decodes a set of bounding boxes into
    the representation used for training the regressors.
    """

    def __init__(self, weights, bbox_xform_clip=math.log(1000. / 16)):
        # type: (Tuple[float, float, float, float], float) -> None
        """
        Arguments:
            weights (4-element tuple)
            bbox_xform_clip (float)
        """
        # RPN中为[1,1,1,1], fastrcnn中为[10,10,5,5]
        self.weights = weights
        # 回归高宽参数的上界
        self.bbox_xform_clip = bbox_xform_clip

    # 解码
    def decode(self, rel_codes, boxes):
        # type: (Tensor, List[Tensor]) -> Tensor
        """
        Args:
            rel_codes: 边界框回归参数
            boxes: 锚点框坐标(左上、右下)
        Returns:
            pred_boxes: 边界框坐标数据
        """
        assert isinstance(boxes, (list, tuple))
        assert isinstance(rel_codes, torch.Tensor)
        # 每张图片的锚点个数。由于图片尺寸是不定的，即特征图尺寸不固定，因此锚点数量也是不定的
        boxes_per_image = [b.size(0) for b in boxes]
        # 将一个batch里的所有锚点boxes拼接在一起，concat_boxes尺寸:[图片中锚点总数*batch, 4]
        concat_boxes = torch.cat(boxes, dim=0)
        # 求锚点总数，也可以直接调取concat_boxes.shape[0]
        box_sum = 0
        for val in boxes_per_image:
            box_sum += val
        # 将预测的bbox回归参数应用到对应anchors上得到预测bbox的坐标。
        # 注意，预测的都是回归参数，需要进一步结合锚点坐标框得到预测的坐标
        pred_boxes = self.decode_single(
            rel_codes, concat_boxes
        )

        # 防止pred_boxes为空时导致reshape报错
        if box_sum > 0:
            pred_boxes = pred_boxes.reshape(box_sum, -1, 4)
        # 返回边界框坐标
        return pred_boxes

    # 解码函数
    def decode_single(self, rel_codes, boxes):
        """
        利用预测的回归参数与锚点做运算，得到最终物体真实的边界框，相当于一个解码操作
        首先将回归参数信息与锚点信息转化成中心坐标信息和宽高信息，之后再做解码，
        得到真实边界框的中心点和宽高，最后再转化为左上角和右下角两个坐标信息
        Arguments:
            rel_codes (Tensor): encoded boxes (边界框回归参数)
            boxes (Tensor): reference boxes (anchors/proposals锚点信息)
        """
        boxes = boxes.to(rel_codes.dtype)

        # xmin, ymin, xmax, ymax
        # anchors/proposals宽度
        widths = boxes[:, 2] - boxes[:, 0]
        # anchors/proposals高度
        heights = boxes[:, 3] - boxes[:, 1]
        # anchors/proposals中心x坐标
        ctr_x = boxes[:, 0] + 0.5 * widths
        # anchors/proposals中心y坐标
        ctr_y = boxes[:, 1] + 0.5 * heights
        # RPN中为[1,1,1,1], fastrcnn中为[10,10,5,5]
        wx, wy, ww, wh = self.weights
        # 预测anchors/proposals的中心坐标x回归参数
        dx = rel_codes[:, 0::4] / wx
        # 预测anchors/proposals的中心坐标y回归参数
        dy = rel_codes[:, 1::4] / wy
        # 预测anchors/proposals的宽度回归参数
        dw = rel_codes[:, 2::4] / ww
        # 预测anchors/proposals的高度回归参数
        dh = rel_codes[:, 3::4] / wh

        # limit max value, prevent sending too large values into torch.exp()
        # self.bbox_xform_clip=math.log(1000. / 16)   4.135
        # 限制上限
        dw = torch.clamp(dw, max=self.bbox_xform_clip)
        dh = torch.clamp(dh, max=self.bbox_xform_clip)
        # 利用公式计算边界框的中心坐标以及宽高
        pred_ctr_x = dx * widths[:, None] + ctr_x[:, None]
        pred_ctr_y = dy * heights[:, None] + ctr_y[:, None]
        pred_w = torch.exp(dw) * widths[:, None]
        pred_h = torch.exp(dh) * heights[:, None]
        # 得到左上角和右下角，这里新创建了0.5，设备和数据类型需一致
        # xmin 
        pred_boxes1 = pred_ctr_x - torch.tensor(0.5, dtype=pred_ctr_x.dtype, device=pred_w.device) * pred_w
        # ymin
        pred_boxes2 = pred_ctr_y - torch.tensor(0.5, dtype=pred_ctr_y.dtype, device=pred_h.device) * pred_h
        # xmax
        pred_boxes3 = pred_ctr_x + torch.tensor(0.5, dtype=pred_ctr_x.dtype, device=pred_w.device) * pred_w
        # ymax
        pred_boxes4 = pred_ctr_y + torch.tensor(0.5, dtype=pred_ctr_y.dtype, device=pred_h.device) * pred_h
        # 将所有的边界框坐标点合并
        pred_boxes = torch.stack((pred_boxes1, pred_boxes2, pred_boxes3, pred_boxes4), dim=2).flatten(1)
        # 返回所有的边界框坐标点
        return pred_boxes

    # 编码
    def encode(self, reference_boxes, proposals):
        # type: (List[Tensor], List[Tensor]) -> List[Tensor]
        """
        结合anchors和与之对应的gt计算regression参数
        Args:
            reference_boxes: List[Tensor] 每个proposal/anchor对应的gt_boxes
            proposals: List[Tensor] anchors/proposals

        Returns: 每个锚点对应的回归参数regression parameters 

        """
        # 统计每张图像的anchors个数，方便后面拼接在一起处理后在分开
        # reference_boxes和proposal数据结构相同
        boxes_per_image = [len(b) for b in reference_boxes]
        reference_boxes = torch.cat(reference_boxes, dim=0)  # 每个锚点对应的目标边界框合并
        proposals = torch.cat(proposals, dim=0)  # 锚点合并

        # 这里相当于解码的逆运算
        targets = self.encode_single(reference_boxes, proposals)
        # 分离，分成batch
        return targets.split(boxes_per_image, 0)  

    # 编码函数
    def encode_single(self, reference_boxes, proposals):
        """
        对每个锚点上的边界框进行编码，得到对应的回归参数
        Arguments:
            reference_boxes (Tensor): 每个锚点对应的目标边界框
            proposals (Tensor): 锚点
        """
        dtype = reference_boxes.dtype
        device = reference_boxes.device
        # 转化成tensor，数据类型与设备类型都相同
        # RPN中为[1, 1, 1, 1], fastrcnn中为[10, 10, 5, 5]
        weights = torch.as_tensor(self.weights, dtype=dtype, device=device)
        targets = encode_boxes(reference_boxes, proposals, weights)

        return targets

    其中encode_boxes函数为：

    def encode_boxes(reference_boxes, proposals, weights):
    # type: (torch.Tensor, torch.Tensor, torch.Tensor) -> torch.Tensor
    """
    利用锚点信息和边界框信息编码得到回归参数
    在公式中参与计算的为矩形框的宽、高、中心坐标
    因此需要先做转换，然后

    Arguments:
        reference_boxes (Tensor): reference boxes(gt)锚点参考的边界框
        proposals (Tensor): boxes to be encoded(anchors)锚点
        weights: 权重
    """

    # perform some unpacking to make it JIT-fusion friendly
    wx = weights[0]
    wy = weights[1]
    ww = weights[2]
    wh = weights[3]
    # 锚点的坐标值
    proposals_x1 = proposals[:, 0].unsqueeze(1) 
    proposals_y1 = proposals[:, 1].unsqueeze(1)
    proposals_x2 = proposals[:, 2].unsqueeze(1)
    proposals_y2 = proposals[:, 3].unsqueeze(1)
    # 用于参考的目标边界框坐标值
    reference_boxes_x1 = reference_boxes[:, 0].unsqueeze(1)  
    reference_boxes_y1 = reference_boxes[:, 1].unsqueeze(1)
    reference_boxes_x2 = reference_boxes[:, 2].unsqueeze(1)
    reference_boxes_y2 = reference_boxes[:, 3].unsqueeze(1)

    # 计算锚点宽、高、中心坐标
    ex_widths = proposals_x2 - proposals_x1  
    ex_heights = proposals_y2 - proposals_y1

    ex_ctr_x = proposals_x1 + 0.5 * ex_widths 
    ex_ctr_y = proposals_y1 + 0.5 * ex_heights
    # 与锚点一样，计算参考边界框宽、高、中心坐标
    gt_widths = reference_boxes_x2 - reference_boxes_x1
    gt_heights = reference_boxes_y2 - reference_boxes_y1
    gt_ctr_x = reference_boxes_x1 + 0.5 * gt_widths
    gt_ctr_y = reference_boxes_y1 + 0.5 * gt_heights
    # 利用公式得到回归参数，分别对应于中心坐标、宽、高
    # 因此最终预测时得到的回归参数也是对应于中心坐标、宽、高
    targets_dx = wx * (gt_ctr_x - ex_ctr_x) / ex_widths
    targets_dy = wy * (gt_ctr_y - ex_ctr_y) / ex_heights
    targets_dw = ww * torch.log(gt_widths / ex_widths)
    targets_dh = wh * torch.log(gt_heights / ex_heights)
    # 将四个回归参数合并
    targets = torch.cat((targets_dx, targets_dy, targets_dw, targets_dh), dim=1)
    return targets
