    域适应目标检测：ICR-CCR¶

    综述¶

    会议与时间：IEEE Conference on Computer Vision and Pattern Recognition 2020 (CVPR, 2020)

    论文地址：http://openaccess.thecvf.com/content_CVPR_2020/papers/Xu_Exploring_Categorical_Regularization_for_Domain_Adaptive_Object_Detection_CVPR_2020_paper.pdf

    源码地址：https://github.com/Megvii-Nanjing/CR-DA-DET

    针对领域：域适应目标检测

    这里以在Domain Adaptive Faster R-CNN算法中引入正则化器为例，网络结构路径：

    CR-DA-DET-master\DA_Faster_ICR_CCR\lib\model\roi_align\faster_rcnn.py

    多标签分类器¶

    这里其实就相当于一个全连接层，用于预测类别分数

    # 相对于原始Faster R-CNN，新定义了两个层
# 全局平均池化，用于将特征图拉伸成一个向量
self.avg_pool = nn.AdaptiveAvgPool2d(output_size=(1, 1))
# 1*1的卷积，用于预测类别分数，与nn.Linear作用一致
self.conv_lst = nn.Conv2d(self.dout_base_model, self.n_classes - 1, 1, 1, 0)

    计算分类器的标签cls_lb，代码位于：

    CR-DA-DET-master\DA_Faster_ICR_CCR\lib\roi_da_data_layer\minibatch.py

    # 这里传入的array为源域数据中的对象类别标签
def gt_classes2cls_lb_onehot(array):
    # 先初始化标签
    cls_lb = np.zeros((num_classes - 1,), np.float32)
    # 遍历所有的对象类别
    for i in array:
        # 利用索引对cls_lb赋值
        # 由于对象类别是从1开始的(0表示背景)，因此多标签分类器的预测类别数量要比ROI预测实例的类别数量少1
        # 对应索引要减1
        cls_lb[i - 1] = 1
    return cls_lb

    ICR模块¶

    因为只有源域数据有图像类别标签，因此ICR损失只针对源域数据

    # 将源域数据传入特征提取网络，提取特征
base_feat = self.RCNN_base(im_data)
# 将源域中的图像特征传入多标签分类器预测分数，用于计算ICR损失
cls_feat = self.conv_lst(self.avg_pool(base_feat)).squeeze(-1).squeeze(-1)
# BCEWithLogitsLoss和二元交叉熵损失函数差不多，只是多了一个sigmoid，即输入先经过sigmoid，再计算损失
# 这里im_cls_lb表示标签y^c
# 计算源域图像中ICR正则化器损失
img_cls_loss = nn.BCEWithLogitsLoss()(cls_feat, im_cls_lb)

    CCR模块¶

    前置工作

      这里只列出了主要的代码，注意：代码中带有前置tgt的变量表示对应于目标域数据的变量，如tgt_base_feat表示由目标域图像生成的特征图，而base_feat表示由源域图像生成的特征图。

    # 将目标域数据传入特征提取网络提取特征
tgt_base_feat = self.RCNN_base(tgt_im_data)
# 将目标域图像特征传入多标签分类器，得到的预测分数用于计算CCR损失
tgt_image_cls_feat = (
    self.conv_lst(self.avg_pool(tgt_base_feat)).squeeze(-1).squeeze(-1)
)
# 将分数经过sigmoid函数
tgt_image_cls_feat = F.sigmoid(tgt_image_cls_feat[0]).detach()

# RPN模块
self.RCNN_rpn.eval()
(
    tgt_rois,
    tgt_rpn_loss_cls,
    tgt_rpn_loss_bbox,
    tgt_rois_binary_score,
) = self.RCNN_rpn(
    tgt_base_feat, tgt_im_info, tgt_gt_boxes, tgt_num_boxes, need_cls_score=True
)
# ROI Pooling模块
tgt_pooled_feat = self.RCNN_roi_crop(
                tgt_base_feat, Variable(tgt_grid_yx).detach()
             )
if cfg.CROP_RESIZE_WITH_MAX_POOL:
    tgt_pooled_feat = F.max_pool2d(tgt_pooled_feat, 2, 2)
tgt_pooled_feat = self._head_to_tail(tgt_pooled_feat)

# 将得到的实例区域特征传入检测器中的分类网络，用于预测类别分数
tgt_cls_score = self.RCNN_cls_score(tgt_pooled_feat).detach()
# 将预测分数传入softmax，得到预测概率
tgt_prob = F.softmax(tgt_cls_score, 1).detach()
# 提取出预测概率值最大的索引，看作该实例的类别
tgt_pre_label = tgt_prob.argmax(1).detach()

    损失的计算

      CCR损失主要是计算二元交叉熵损失的权重d_j，并且只针对目标域数据的前景实例，对于源域数据和目标域数据中的背景实例，d_j均设置为1。

    # 这里表示权重d_j
target_weight = []
#  按预测到的实例(目标)进行遍历
for i in range(len(tgt_pre_label)):
    # 得到第i个目标的预测标签
    label_i = tgt_pre_label[i].item()
    # 0表示背景，如果该物体非背景的话，就计算d_j
    if label_i > 0:
        # 这里表示计算d_j，对应论文公式(5)
        # tgt_image_cls_feat表示多标签分类器预测的分数，对应论文里的p_j
        # tgt_prob表示ROI Head对实例的分类结果
        diff_value = torch.exp(
            weight_value
            * torch.abs(tgt_image_cls_feat[label_i - 1] - tgt_prob[i][label_i])
        ).item()
        # 存储得到的d_j
        target_weight.append(diff_value)
    # 如果该物体是背景的话，则d_j=0，即权重不变
    else:
        target_weight.append(1.0)
# 初始化二元交叉熵损失函数，将上面得到的权重d_j放进去
tgt_instance_loss = nn.BCELoss(
    weight=torch.Tensor(target_weight).view(-1, 1).cuda()
)
# 计算目标域图像中，引入了CCR的实例级对抗损失
tgt_DA_ins_loss_cls = tgt_instance_loss(
    tgt_instance_sigmoid, tgt_same_size_label
)
