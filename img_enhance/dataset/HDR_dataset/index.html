    Kalantari¶

    官方下载地址：https://cseweb.ucsd.edu/~viscomp/projects/SIG17HDR

    输入图像：.tif格式，48位深度（48表示16*3），三个通道数据范围为0 \sim 2^{16}-1，即0 ~ 65535；

    标签图像：.hdr格式，读取的图像数据范围为0.0\sim 1.0，浮点数据类型，越接近于1，表示亮度越大。

    训练过程：会将输入的低动态范围图归一化，之后转为高动态范围图，转换公式：(imgs ** gamma) / (expo + 1e-8)，gamma超参数在论文《HDR-Transformer》中设置为2.2，expo表示曝光时间（或曝光值）（可以补偿曝光带来的亮度变化，使得图像中的亮度更加准确。），该数据储存在exposure.txt。

    # LDR转为HDR，数据范围为0-1
def ldr_to_hdr(imgs, expo, gamma=2.2):
    return (imgs ** gamma) / (expo + 1e-8)

    注：在训练过程中，通常将一张图的ldr图与对应的hdr图数据合并，同时当做网络的一个输入。网络最后一层为sigmoid，将输出的图片做归一化，之后将读到hdr图像做L1损失。（所读取的图像范围就是0-1，不需要额外做归一化处理）

    测试过程：直接将网络的输出视为hdr图像，以.hdr图像的格式保存即可，评估指标的时候也是直接利用模型的输入输出做评估。

    save_hdr(os.path.join(args.save_dir, '{}_pred.hdr'.format(idx)), pred_hdr)

# 将图像数据转存为.hdr格式的文件
def radiance_writer(out_path, image):
    with open(out_path, "wb") as f:
        # 向文件中写入一些头部信息，以注释的形式标识RADIANCE格式
        # 并指定图像的格式为32-bit RLE (Run Length Encoded) RGBE。
        f.write(b"#?RADIANCE\n# Made with Python & Numpy\nFORMAT=32-bit_rle_rgbe\n\n")
        # 写入图像的大小信息，指定图像的高度（Y轴方向）和宽度（X轴方向）。
        f.write(b"-Y %d +X %d\n" % (image.shape[0], image.shape[1]))
        # 计算图像中最亮的像素值，并使用np.frexp函数将最亮像素值拆分为尾数和指数的形式。
        # 然后通过缩放尾数将其映射到0到255的范围内。
        brightest = np.maximum(np.maximum(image[..., 0], image[..., 1]), image[..., 2])
        mantissa = np.zeros_like(brightest)
        exponent = np.zeros_like(brightest)
        np.frexp(brightest, mantissa, exponent)
        scaled_mantissa = mantissa * 255.0 / brightest
        # 创建一个新的数组rgbe，用于存储调整后的图像数据。
        # 将图像的RGB通道乘以尾数并四舍五入为整数，然后将指数加上128。
        rgbe = np.zeros((image.shape[0], image.shape[1], 4), dtype=np.uint8)
        rgbe[..., 0:3] = np.around(image[..., 0:3] * scaled_mantissa[..., None])
        rgbe[..., 3] = np.around(exponent + 128)
        # 将rgbe数组展平并以二进制形式写入到打开的文件中。
        rgbe.flatten().tofile(f)

    代码引自：https://github.com/liuzhen03/HDR-Transformer-PyTorch

    NTIRE 2021/2022¶

    下载地址：https://codalab.lisn.upsaclay.fr/competitions/1515

    输入图像：.png格式，24位深度（8*3），三个通道数据范围为0 \sim 2^8-1，即0 \sim 255；

    标签图像：.png格式，48位深度（16*3），三个通道数据范围为0 \sim 2^{16}-1，即0 \sim 65535，在读取的时候，需要做额外的处理，恢复HDR图像的原始范围（比赛官网数据介绍中写到的）

    ​ 输入图像读取后直接做归一化操作，转为浮点数据类型。标签图像读取后利用比赛官方提供的代码，将图像恢复成HDR图像的范围（这里的图像数据并不是0-1之间，曝光度高的地方数据值会大于1，有点奇怪，和上面Kalantari数据中的.hdr图像数据格式不太一样）

    # NTIRE官方给的读取HDR图像的代码（读取标签图像）
def imread_uint16_png(image_path, alignratio_path):
    """ This function loads a uint16 png image from the specified path and restore its original image range with
    the ratio stored in the specified alignratio.npy respective path.


    Args:
        image_path (str): Path to the uint16 png image
        alignratio_path (str): Path to the alignratio.npy file corresponding to the image

    Returns:
        np.ndarray (np.float32, (h,w,3)): Returns the RGB HDR image specified in image_path.

    """
    # Load the align_ratio variable and ensure is in np.float32 precision
    align_ratio = np.load(alignratio_path).astype(np.float32)
    # Load image without changing bit depth and normalize by align ratio
    return cv2.cvtColor(cv2.imread(image_path, cv2.IMREAD_UNCHANGED), cv2.COLOR_BGR2RGB) / align_ratio
